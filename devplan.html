1. Relational Schema

Our schema models events, venues, tickets, users, and registrations. Keys and data types are defined below, and we include a visual ER diagram for clarity.

Venue
  venue_id INT PK
  name VARCHAR(150)
  address VARCHAR(255)
  rating DECIMAL(2,1)

Event
  event_id INT PK
  title VARCHAR(200)
  description TEXT
  start_date DATE
  start_time TIME
  venue_id INT FK → Venue.venue_id
  link VARCHAR(500)
  source VARCHAR(100)

Ticket
  ticket_id INT PK
  event_id INT FK → Event.event_id
  provider VARCHAR(100)
  ticket_link VARCHAR(500)

User
  user_id INT PK
  name VARCHAR(150)
  email VARCHAR(200)
  password VARCHAR(200)

Registration
  reg_id INT PK
  user_id INT FK → User.user_id
  event_id INT FK → Event.event_id
  timestamp DATETIME

Functional Dependencies:

  • venue_id → name, address, rating
  • event_id → title, description, start_date, start_time, venue_id, link, source
  • ticket_id → event_id, provider, ticket_link
  • user_id → name, email, password
  • reg_id → user_id, event_id, timestamp

2. Databases and Platforms

We will use MySQL on db8.cse.nd.edu as our primary relational database.
Our backend will be built in Python (Flask) to provide REST endpoints and handle database operations.
PHP may be used selectively for compatibility with the server.
The frontend will use HTML, CSS, JavaScript, and Bootstrap for responsive, user-friendly design.

3. Data Sources

We collect events dynamically using the SerpAPI Google Events API. Our Python script:

  • Queries SerpAPI for “Events near South Bend.”
  • Paginates results (currently ~500 unique events collected).
  • Deduplicates events by (link, start_date).
  • Saves all unique events into southbend_events.json.

Scaling Strategy: To meet the Stage 3 requirement of 10,000+ records across 5+ relations, we will:

  • Expand queries to multiple cities (Chicago, Indianapolis, Detroit, etc.).
  • Generate additional synthetic test data using Python’s Faker library for users, registrations, and venues.
  • Supplement event data with secondary APIs (Eventbrite, Ticketmaster) if necessary.
